## 项目整体目录结构
```text
/
├── client/          # 前端文件夹
├── server/          # 后端文件夹
```

---

## 核心规则与约束

### 1. 前端目录
- 路径：`client/`
- 说明：存放所有前端代码与资源文件

---

### 2. 后端目录（`server/`）强制要求
后端语言不做限制，但必须满足以下文件与接口规范：

#### 2.1 必须包含的文件
| 文件路径            | 作用说明                                                               |
| ------------------- | ---------------------------------------------------------------------- |
| `server/Dockerfile` | Docker 构建配置文件，用于构建后端镜像                                  |
| `server/build.sh`   | 构建脚本，负责初始化依赖安装、编译打包，并为 Dockerfile 准备好构建产物 |

#### 2.2 后端服务与接口强制规范
1.  **服务端口**：必须监听 **8888** 端口
2.  **API 路径前缀**：所有 API 接口必须以 `/api/` 开头
3.  **必须实现健康检查接口**：
    - 路径：`/api/health`
    - 作用：用于平台进行服务状态检测
