# 统一任务中心

`hooks/task` 提供系统任务的执行能力和统一任务中心查询。文件处理任务由 `hooks/file/tasks/file-processing` 负责执行，只把通用状态和查询字段写入 `tasks`。

```text
系统脚本任务 ─────┐
                  ├── tasks ─▶ /sys/task/* ─▶ 管理端任务中心
文件处理任务 ─────┘
```

## 数据职责

- `tasks`：任务状态唯一事实来源，保存分类、业务对象、阶段、进度、错误和执行时间。
- `file_processing_tasks`：文件、文档版本、知识库、执行序号、触发来源和结果摘要。
- `file_processing_task_stage_runs`：阶段尝试、checkpoint、数量、耗时和错误。

任务中心列表与状态统计必须使用同一筛选条件和可见范围：普通用户查看自己发起的任务，知识库管理员查看当前租户任务，系统管理员查看全部任务。技术日志使用独立权限，日志不得包含完整文件内容、长期对象地址或存储凭证。

文件任务取消、失败重试和再次执行由文件域服务处理；系统脚本任务继续使用现有停止与调度能力。任务中心只负责统一查询和操作入口，不直接执行文件解析或知识库逻辑。
