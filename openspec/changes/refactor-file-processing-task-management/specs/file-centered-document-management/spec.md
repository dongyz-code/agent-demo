## ADDED Requirements

### Requirement: 单一文件管理入口
管理端 MUST 使用单一文件管理页面承载文件上传、筛选、预览、下载、删除、开始处理、重新处理和查看任务。管理端 MUST NOT 将通用文件和上传会话作为独立业务页签展示。

#### Scenario: 用户查看文件管理
- **WHEN** 授权用户进入文件管理页面
- **THEN** 系统展示文件列表及其最新处理状态，而不是要求用户在通用文件、上传会话和 RAG 文档页面之间切换

#### Scenario: 上传过程恢复
- **WHEN** 文件采用分片上传并发生暂停或失败
- **THEN** 用户在上传弹窗内查看并恢复上传，任务中心和文件页面不新增独立上传会话记录

### Requirement: 上传时选择是否进入 RAG
文件上传入口 MUST 提供是否进入 RAG 的选择，并根据服务端配置设置默认值。选择进入 RAG 时 MUST 指定有效目标知识库；未选择时 MUST 只保存已验证文件。

#### Scenario: 上传后自动处理
- **WHEN** 用户上传文件并选择进入 RAG且目标知识库有效
- **THEN** 文件验证成功后系统自动创建一个文件处理任务

#### Scenario: 仅上传文件
- **WHEN** 用户上传文件但未选择进入 RAG
- **THEN** 系统保存文件并将其展示为未处理，不自动创建文件处理任务

#### Scenario: 目标知识库不可用
- **WHEN** 用户选择进入 RAG但目标知识库不存在、已停用或无权访问
- **THEN** 系统拒绝处理意图并提供可理解的知识库错误，不创建文件处理任务

### Requirement: 上传意图可靠持久化
系统 MUST 在服务端持久化上传时的 RAG 处理意图，并在文件验证成功后幂等创建任务，不得依赖浏览器上传完成后的额外串联调用保证任务创建。

#### Scenario: 上传完成后浏览器离线
- **WHEN** 文件对象已上传并验证成功但浏览器在完成响应前离线
- **THEN** 系统仍按照已保存意图创建且只创建一个文件处理任务

#### Scenario: 重复完成请求
- **WHEN** 客户端重复提交同一上传完成请求
- **THEN** 系统返回同一文件结果且不重复创建处理任务

### Requirement: 手动开始文件处理
未进入 RAG 的已验证文件 MUST 可以从文件列表手动创建处理任务，并在创建时选择目标知识库。

#### Scenario: 手动处理未处理文件
- **WHEN** 用户对未处理文件点击开始处理并选择有效知识库
- **THEN** 系统创建该文件的首次处理任务并在文件列表展示等待执行状态

### Requirement: 文件列表聚合任务摘要
文件列表 MUST 一次返回文件基础信息、是否进入 RAG、当前活动任务、累计执行次数、最后成功任务和知识库摘要，不得要求管理端逐行请求多个接口拼装状态。

#### Scenario: 文件存在历史和活动任务
- **WHEN** 文件已有两次历史任务且第三次任务正在执行
- **THEN** 文件列表展示执行次数为三、当前任务阶段和最后一次成功任务摘要

#### Scenario: 文件从未处理
- **WHEN** 文件没有任何处理任务
- **THEN** 文件列表展示执行次数为零且处理状态为未处理

### Requirement: 多文件独立处理
多文件上传选择进入 RAG 时，系统 MUST 为每个验证成功的文件创建独立任务，并允许任务并行执行。单个文件失败 MUST NOT 阻止其他文件继续执行。

#### Scenario: 多文件部分失败
- **WHEN** 用户上传三个文件且其中一个文件处理失败
- **THEN** 另外两个文件继续执行并分别保存自己的任务结果

### Requirement: 文件与任务导航
文件管理页面 MUST 支持从文件行查看该文件的任务历史，并将文件筛选条件传递给统一任务中心。

#### Scenario: 查看指定文件任务
- **WHEN** 用户点击某个文件的查看任务操作
- **THEN** 系统打开现有任务中心并只展示用户有权查看的该文件任务
