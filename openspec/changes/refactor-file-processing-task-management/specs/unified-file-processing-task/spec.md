## ADDED Requirements

### Requirement: 一个文件一次执行对应一个任务
系统 MUST 将一个文件的一次完整预处理与 RAG 接入记录为一个文件处理任务。读取、解析、标准化、切分和 RAG 接入 MUST 作为该任务的阶段记录，不得在统一任务中心中平铺为多条一级任务。

#### Scenario: 完整文件处理
- **WHEN** 文件处理任务正常执行
- **THEN** 任务依次记录预处理和 RAG 接入阶段，并最终只产生一条成功任务记录

#### Scenario: 阶段详情
- **WHEN** 用户查看文件处理任务详情
- **THEN** 系统在同一任务中展示各阶段状态、耗时、进度和错误

### Requirement: 任务状态与当前阶段分离
文件处理任务 MUST 独立记录任务状态和当前阶段。任务状态 MUST 至少支持等待执行、执行中、执行成功、执行失败和已取消；当前阶段 MUST 表示任务正在执行的具体步骤。

#### Scenario: 正在切分文件
- **WHEN** 任务正在执行文档切分阶段
- **THEN** 任务状态展示为执行中且当前阶段展示为文档切分

#### Scenario: 解析失败
- **WHEN** 文件在解析阶段发生不可恢复错误
- **THEN** 任务状态展示为执行失败并保留解析阶段、稳定错误码和用户可理解的错误信息

### Requirement: 执行次数与历史不可变
系统 MUST 为同一文件的任务分配递增执行序号。重新执行和失败重试 MUST 创建新的任务记录，不得覆盖历史任务状态、结果或错误。

#### Scenario: 失败后重试
- **WHEN** 文件第二次执行失败且用户点击重试
- **THEN** 系统保留第二次失败记录并创建执行序号为三的新任务

#### Scenario: 查看执行次数
- **WHEN** 文件已创建三次处理任务
- **THEN** 文件摘要和任务详情均能显示累计执行三次及每次结果

### Requirement: 防止等价活动任务重复
同一租户内，相同文件、目标知识库和处理配置 MUST 最多存在一个等待执行或执行中的活动任务。

#### Scenario: 重复点击开始处理
- **WHEN** 用户对已有活动任务的相同文件和知识库再次点击开始处理
- **THEN** 系统返回现有活动任务并引导用户查看，不创建重复任务

#### Scenario: 不同知识库处理
- **WHEN** 同一文件需要进入另一个有效知识库
- **THEN** 系统允许创建目标知识库不同的处理任务

### Requirement: 任务取消
等待执行或执行中的文件任务 MUST 支持取消。取消 MUST 在安全阶段边界停止后续工作，并且不得删除此前成功任务和已发布结果。

#### Scenario: 取消等待任务
- **WHEN** 用户取消尚未开始的任务
- **THEN** 系统将任务标记为已取消且不执行任何预处理阶段

#### Scenario: 取消执行中任务
- **WHEN** 用户取消正在执行的任务
- **THEN** 系统在当前可安全中止点停止后续阶段并保留已记录的阶段结果用于审计

### Requirement: 持久化执行与恢复
文件处理任务 MUST 通过持久化状态领取和执行，不得仅依赖单个服务进程的内存队列。服务重启后 MUST 能识别未完成任务并恢复或标记为可重试失败。

#### Scenario: 等待任务遇到重启
- **WHEN** 服务在任务仍处于等待执行时重启
- **THEN** 重启后的执行器能够重新领取该任务

#### Scenario: 执行中任务失去心跳
- **WHEN** 执行中任务超过规定时间未更新心跳
- **THEN** 系统根据阶段 checkpoint 恢复执行或以明确错误标记为可重试失败

### Requirement: 并行任务相互隔离
不同文件的任务 MUST 可以并行执行，并使用独立状态、错误和产物。一个任务的失败或取消 MUST NOT 修改其他文件任务的状态。

#### Scenario: 并行任务部分失败
- **WHEN** 三个文件任务并行执行且其中一个解析失败
- **THEN** 失败任务记录自己的错误，另外两个任务继续执行到各自终态

### Requirement: RAG 接入结果准确表达
在尚未实现 Embedding 和检索索引时，文件任务 MUST 将最终阶段描述为 RAG 接入或知识库关联，不得向用户宣称已经完成向量索引。任务成功 MUST 产出版本化 Segment 和有效知识库关联。

#### Scenario: 当前版本处理成功
- **WHEN** 文件完成解析、标准化、切分并成功加入目标知识库
- **THEN** 任务展示为 RAG 接入成功并记录 Segment 数量和知识库信息

#### Scenario: 后续增加索引阶段
- **WHEN** 后续版本启用 Embedding 或索引阶段
- **THEN** 系统通过版本化任务配置和新增阶段记录执行，不静默改变旧任务含义

### Requirement: 每种任务独立目录
服务端 MUST 将每种文件任务定义放在 `hooks/file/tasks/<task-key>/` 独立目录，并通过文件域公共入口和通用任务接口交互。任务目录 MUST 收口定义、runner、阶段和任务专属类型，不得跨目录导入其他任务内部实现。

#### Scenario: 新增文件任务类型
- **WHEN** 开发者新增一种文件任务
- **THEN** 其实现位于独立任务目录并通过注册或公共入口接入任务中心，不修改其他任务内部文件
