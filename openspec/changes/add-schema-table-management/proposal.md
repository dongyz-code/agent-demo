## Why

当前系统的表结构变更依赖开发者手动运行 Drizzle 迁移，缺少一个面向有权限管理员的可视化管理入口。新增表管理能力后，管理员可以基于仓库内 Drizzle schema 查看可管理表、预览数据、执行受控重命名，并在需要时通过无损重建流程把数据库表结构校准到 schema。

## What Changes

- 在管理端新增系统表管理页面，用于查看当前用户有权限访问的 Drizzle schema 表、字段结构、索引/约束摘要和 demo 数据。
- 新增 `/sys/table/*` 后端接口，提供表清单、表详情、数据预览、重命名预演/执行、schema 重置预演/执行能力。
- 新增表管理权限点，区分查看、数据预览、结构重命名、结构重置等高风险操作。
- 新增基于 Drizzle schema 的结构解析层，只暴露 `schemaTables` 注册的业务表，默认排除迁移表、临时表和未注册数据库表。
- 新增结构变更执行器：对重命名和 schema 重置先生成计划，再由管理员确认执行；执行过程写入用户操作日志。
- schema 重置采用无损优先流程：创建临时新表、按可兼容字段复制数据、校验行数和关键约束、交换表名、保留备份表，失败时回滚。
- 不引入自动后台迁移，不在页面外静默修改数据库结构；所有 DDL 操作必须显式确认。

## Capabilities

### New Capabilities

- `schema-table-management`: 基于 Drizzle schema 的表管理能力，覆盖有权限表清单、结构查看、demo 数据预览、表/字段重命名和无损 schema 重置。

### Modified Capabilities

无。

## Impact

- 后端：新增数据库元数据服务、DDL 计划生成与执行服务、`/sys/table/*` 路由、权限判断、操作日志。
- 类型包：新增表管理请求/响应类型、权限 key 相关类型。
- 管理端 Vue admin：新增表管理路由、菜单、权限树、页面、操作抽屉/对话框、预演结果视图和二次确认流程。
- 数据库：新增可选的表结构操作记录表，用于审计、状态追踪和失败恢复；不会直接修改现有业务表数据模型。
- 运维：高风险 DDL 操作需要数据库事务、会话级 advisory lock、超时控制和可观测日志。
